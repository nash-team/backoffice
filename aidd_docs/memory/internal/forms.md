# Forms

## Approach

- No form library (no React Hook Form, Formik)
- Manual state with `useState`
- Simple forms, no complex validation needs

## Pattern

```typescript
const [form, setForm] = useState({ theme: '', audience: '', num_pages: 8 })

await dispatch(createEbook({ ...form })).unwrap()
```

## Validation

- **Client-side**: HTML constraints (`required`, `min`, `max`)
- **Button disabling**: `disabled={!form.theme || !form.audience}`
- **No schema validation** (no Zod, Yup)
- **Server-side**: Backend validates via Pydantic

## Forms

### CreateEbookModal
- Fields: `theme` (select), `audience` (select), `num_pages` (number), `preview_mode` (checkbox)
- Title is auto-generated by the backend (no title field)
- Config loaded via `fetchFormConfig()` thunk on modal open
- Options from backend `GET /api/ebooks/form-config`
- Theme/audience options support both string and object formats (`ThemeOption`/`AudienceOption` with `id`/`label`)
- Submit -> `createEbook` thunk -> refreshes list + stats
- Resets on close

### PageEditModal
- Fields: `editPrompt` (textarea)
- Two modes: Regenerate (new image from prompt) and Correct (targeted edit)
- Workflow: load page data → regenerate/edit preview → undo if needed → apply
- Four actions: Preview Regenerate, Preview Edit, Undo (restorePreview), Apply (save to DB)
- Circular progress spinner with percentage during generation
- Image displayed as base64 `<img src="data:image/png;base64,...">`
